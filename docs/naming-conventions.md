# Naming conventions

Two independent surfaces enforce filename rules:

1. **The slug rule** — every folder segment + filename in the vault
   must be lowercase-kebab-case. Enforced uniformly across `.md` and
   asset files. Hardcoded in plugin code (it's pure infrastructure
   that can't be expressed as a template rule).
2. **`namingPatterns`** — per-folder filename regexes declared in
   `.claude/vault-keeper.json`. Optional, vault-specific.

For folder placement enforced by individual templates, see
[templates/folder-and-naming](templates/folder-and-naming.md#allowed_folders).

## The slug rule

### What it enforces

Each path segment (folder OR file basename) must match:

```
[a-z0-9]+(-[a-z0-9]+)*
```

Plain English:

- Lowercase ASCII letters + digits + hyphen only.
- No leading or trailing hyphen.
- No consecutive hyphens (`--`).
- No spaces, no underscores, no uppercase, no diacritics.

### Filenames split on the first dot

Filenames are split on the **first** `.` — the name portion follows
the slug rule, and each subsequent extension segment also follows the
slug rule.

| Filename | Name | Ext chain | Verdict |
|---|---|---|---|
| `note.md` | `note` | `md` | ✅ |
| `tailwind.config.js` | `tailwind` | `config.js` | ✅ (both `config` and `js` are slug-shape) |
| `Note.md` | `Note` | — | ❌ uppercase |
| `note 1.md` | `note 1` | — | ❌ space |
| `note_one.md` | `note_one` | — | ❌ underscore |
| `note.md.bak` | `note` | `md.bak` | ✅ (`bak` is slug-shape) |
| `note.Bad.md` | `note` | `Bad.md` | ❌ (uppercase `Bad` segment) |

### Task-ID prefix exception

Files whose name portion matches the task-id shape are accepted
unconditionally:

```
^[a-z]+-\d+-[a-z0-9]+(-[a-z0-9]+)*$
```

Concrete: `t-001-cleanup.md`, `prd-013-feature.md`,
`adr-007-pick-bun.md` all pass. The prefix may be any lowercase
letter sequence; the digit run is the discriminating identifier; the
trailing slug follows the normal kebab-case rule.

### Exempt basenames

Well-known files bypass the slug rule entirely:

```
README.md  BOARD.md  CLAUDE.md  MEMORY.md  LICENSE  LICENSE.md
CHANGELOG.md  CONTRIBUTING.md  DESIGN.md  AGENTS.md
```

These names are conventions of the broader ecosystem:

- `README.md`, `LICENSE`, `CHANGELOG.md`, `CONTRIBUTING.md` — open-
  source norms.
- `BOARD.md` — kanban-view convention.
- `CLAUDE.md` / `CLAUDE.local.md` / `AGENTS.md` — Claude Code +
  multi-agent context conventions.
- `MEMORY.md` — agent persistence convention.
- `DESIGN.md` — Google Labs spec format.

The exempt list is `CONFIG.slug.exemptBasenames` in `lib/validators.js`.
Hidden files (any segment starting with `.`) are also skipped — `.git`,
`.gitkeep`, `.gitignore` etc. are tooling artifacts.

### Error shape

A folder-segment violation:

```
[ERR ]  field=folder
        Folder 'Bad Folder' violates slug convention (lowercase letters,
        digits, hyphens only)
        fix: Rename folder to 'bad-folder/'
```

A filename violation:

```
[ERR ]  field=filename
        Filename 'My Note.md' violates slug convention (lowercase letters,
        digits, hyphens only)
        fix: Rename to 'my-note.md' (or another lowercase-kebab name)
```

An extension-segment violation:

```
[ERR ]  field=filename
        Extension segment 'Bad' in 'note.Bad.md' violates slug convention
        fix: Each dot-separated segment must be lowercase-kebab — e.g.
        'note.bad.md'
```

The fix suggestion is generated by `suggestSlug()` — camelCase →
camel-case, underscores → hyphens, strip non-`[a-z0-9-]`, collapse
double hyphens.

### Suggestion algorithm

```
"TrueProfit tone and voice"   → "trueprofit-tone-and-voice"
"syncStartHandler"            → "sync-start-handler"
"design_tokens"               → "design-tokens"
"SHTP-6138-cogs"              → "shtp-6138-cogs"
"BadFolder"                   → "bad-folder"
```

Steps:

1. Insert `-` between lowercase/digit and uppercase (`camelCase` →
   `camel-Case`).
2. Insert `-` between consecutive uppercase + uppercase-lowercase
   (`ABCFoo` → `ABC-Foo`).
3. Replace runs of underscores / whitespace with `-`.
4. Lowercase everything.
5. Strip any character that isn't `[a-z0-9-]`.
6. Collapse repeated `--`.
7. Trim leading/trailing `-`.

### Where the check runs

Twice in the CLI:

1. Once per markdown document (inside `validateDocument`, calling
   `validateSlug(filepath)`).
2. Once per non-markdown asset (the separate slug-pass over the same
   scan scope), where issues are emitted with `docType: "asset"`.

The LSP runs the markdown branch on every keystroke; asset slugs are
CLI-only.

## `namingPatterns` (vault config)

Per-folder filename regexes declared in `.claude/vault-keeper.json`:

```json
{
  "namingPatterns": {
    "docs/decisions": "^\\d{4}-\\d{2}-\\d{2}-adr-\\d{3}-[a-z0-9-]+\\.md$",
    "docs/notes":     "^note-\\d{3}-[a-z0-9-]+\\.md$"
  }
}
```

### Matching

For each entry, the validator checks whether the document path
**contains** the folder key. If yes, the regex is tested against the
basename. Files **nested deeper** than the folder root (i.e. with
intermediate path segments between the folder and the file) are
exempted — they live inside a bundle or sub-folder and follow that
template's own `allowed_folders`.

`README.md` is always exempt from `namingPatterns` (it's the universal
folder-meta name).

### `namingPatterns` vs template `allowed_folders`

| When to use | Rule kind |
|---|---|
| One template, one filename convention | Template `validation_rules.allowed_folders` |
| One folder, multiple templates sharing one filename convention | Vault `namingPatterns` |

In a vault where one folder hosts only one template type, prefer
template-level `allowed_folders` — keeps the rule co-located with the
template that requires it.

## Putting it together

Example vault layout:

```
docs/
├── notes/
│   ├── note-001-hello.md            ✅
│   ├── note-002-second.md           ✅
│   └── Random Thoughts.md           ❌ slug rule fails
├── decisions/
│   ├── 2026-05-12-adr-007-foo.md    ✅
│   └── random.md                    ❌ namingPatterns fails (no date prefix)
├── images/
│   ├── login-screen.png             ✅
│   └── LoginScreen.png              ❌ slug rule fails (asset slug pass)
└── README.md                        ✅ exempt basename
```

Configuration:

```json
{
  "vaultRoot": "docs",
  "vaultFolders": ["docs"],
  "namingPatterns": {
    "docs/decisions": "^\\d{4}-\\d{2}-\\d{2}-adr-\\d{3}-[a-z0-9-]+\\.md$"
  }
}
```

The slug rule fires on `Random Thoughts.md` and `LoginScreen.png`
unconditionally. `namingPatterns` fires on `decisions/random.md`
because the regex requires the date prefix.
